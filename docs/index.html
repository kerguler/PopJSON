PopJSON

JSON representation of the dynamically-structured matrix population models with multiple processes.

Using the library

Model definition

To define the algorithm and its parameters for simulation. For now, only Population is defined (ODE is pending), which could be either deterministic or stochastic. To set the precision of the accumulative process indicator, istep is used. This effectively limits the maximum number of pseudo-stage classes.

{
    "model": {
            "title": "Climate-sensitive population dynamics of Aedes albopictus",
            "type": "Population",
            "url": "https://github.com/kerguler/Population",
            "deterministic": true,
            "parameters": {
                "algorithm": "Population",
                "istep": 0.01
            }
        }
}

Declaring a population (or a development stage)

{
    "model": {
            "title": "Climate-sensitive population dynamics of Aedes albopictus",
            "type": "Population",
            "url": "https://github.com/kerguler/Population",
            "deterministic": true,
            "parameters": {
                "algorithm": "Population",
                "istep": 0.01
            }
        },
    "populations": [
        {
            "id": "adult",
            "name": "Adult females",
            "processes": [
                {
                    "id": "adult_lifetime",
                    "name": "Adult female lifetime",
                    "arbiter": "ACC_ERLANG",
                    "value": [10, 4]
                }
            ]
        }
    ]
}
